# Blinded Pilot Output Packets

These files are de-identified final outputs prepared for later rubric-based scoring under `rubric.md`.

## Use notes
- Judges should score **final outputs only**.
- Do not reveal condition labels, participant identities, timestamps, or chat transcripts to judges.
- Each output should be paired with the relevant task packet / answer key during scoring.
- For Session 1, raw task-specific point scores remain the primary within-task metrics:
  - `protocol/pilot_task.md` / 650-point seeded-bug rubric
  - `pilot_task_b/task.js` / 525-point seeded-bug rubric
- These blinded packets are intended for a second, cross-task comparable layer using the 0-4 dimensions in `rubric.md`.

## Packet mapping
- `output_A.md` — completed output from one pilot condition (task family: seeded bug inspection)
- `output_B.md` — completed output from one pilot condition (task family: seeded bug inspection)
- `output_C.md` — reserved for the pending solo output

To preserve blinding, the mapping from output IDs to conditions should not be shown to judges.
